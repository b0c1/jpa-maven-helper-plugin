Generaing Elements in persisitence.xml files

  • Summary

    The generate-entities goals scan the the classpath and dependencies jar, looking for JPA entities, mapped superclasses, embeddable and id classes. The plugin collect all classes and add their names to the persistence.xml selected persistence unit. The persistence unit in the test packages visible only in the test persistence.xml.

  • Generate the entities
  • To generate all found entity to all persistence unit +---+ plugin groupIdhu.javaportal.maven.plugin/groupId artifactIdjpa-maven-helper-plugin/artifactId version1.0.0-SNAPSHOT/version executions execution idCollect and write JPA entities to persistence.xml/id goals goalgenerate-entities/goal /goals /execution /executions /plugin +---+
  • Filter package by persistence unit, if you define one or more filter, you need to define ALL persistence unit (the non existing unit will be ignored) +---+ plugin groupIdhu.javaportal.maven.plugin/groupId artifactIdjpa-maven-helper-plugin/artifactId version1.0.0-SNAPSHOT/version executions execution idCollect and write JPA entities to persistence.xml/id goals goalgenerate-entities/goal /goals /execution configuration filters !-- In the persistence unit named 'test2' only the test.sub and test.sub2 package persistence unit exists (plus the manual added <class entries --> filter nametest2/name packages packagetest.sub/package packagetest.sub2/package /packages /filter !-- If no packages defined, all entities will be written -- filter nametest1/name /filter /filters /configuration /executions /plugin +---+
